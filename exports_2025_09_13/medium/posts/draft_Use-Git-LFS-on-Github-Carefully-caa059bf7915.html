Git Large File System is a great solution when you need to store large files (audio files, PDFs, test fixtures, binaries, etc…) as part of your repo. You should probably be using some sort of object storage system when the number of size of the files exceeds some threshold, but it’s a very quick and easy way to get going and can take you far.

I create a new repo called git-lfs-test and confirmed that my Git LFS usage is currently empty using the instructions provided by Github and visiting the billing page.

I then created a 50Mb file, commit it and pushed it. My usage remained at 0.0GB, so I assumed it was likely because GitHub round 0.05GB down.

Since every account only comes with 1GB of alloted Git LFS storage, you need to be very careful because of how the data is versioned.From the docs:

If you push a 500 MB file to Git LFS, you’ll use 500 MB of your allotted storage and none of your bandwidth. If you make a 1 byte change and push the file again, you’ll use another 500 MB of storage and no bandwidth, bringing your total usage for these two pushes to 1 GB of storage and zero bandwidth.
